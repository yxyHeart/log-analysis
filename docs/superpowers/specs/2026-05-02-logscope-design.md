# LogScope — AI日志侦探

## Context

开发者在调试排错时面临多方面痛点：日志大海捞针、根因分析难、线上问题难复现。现有通用AI工具（如ChatGPT、Claude）没有针对日志场景做深度优化。LogScope是一个Web应用，专门解决"给一堆日志，快速定位问题根因"这个高频痛点。

## 核心架构

```
日志输入 → 预处理/结构化 → AI分析 → 结果呈现 → 追问交互
```

### 模块

1. **LogInput** — 日志输入层
   - 粘贴文本、上传文件（.log, .txt, .json）
   - 多段日志拼接（来自不同服务）
   - 自动检测日志来源（框架、语言、格式）

2. **LogParser** — 日志预处理层
   - 自动识别日志格式（JSON/文本/混合），提取时间戳、级别、消息体
   - 按时间排序，去重
   - 识别并标记异常堆栈（stack trace）

3. **LogAnalyzer** — AI分析引擎
   - 阶段1: 错误提取 — 找出所有ERROR/WARN及关联上下文
   - 阶段2: 时间线重建 — 从首条异常信号到最终报错的因果链
   - 阶段3: 根因分析 — 最可能的根因 + 置信度
   - 阶段4: 修复建议 — 可操作的修复方案

4. **ResultView** — 结果呈现层
   - 左侧：原始日志（高亮关键行）
   - 右侧：AI分析结果（时间线、根因、建议）
   - 点击日志行跳转到对应分析

5. **ChatPanel** — 追问面板
   - 基于当前日志上下文的对话
   - 示例："为什么这个请求会失败？" "这个错误和5分钟前那个WARN有关系吗？"

## 技术栈

- **前端**: Next.js (App Router) + TypeScript + Tailwind CSS
- **AI调用**: Vercel AI SDK（统一多模型、流式输出、结构化输出）
- **状态管理**: React内置（useState/useReducer）+ URL state

## 项目结构

```
src/
  app/
    page.tsx              # 主页 - 日志输入
    api/
      analyze/route.ts    # 分析API（流式）
      chat/route.ts       # 追问API（流式）
    result/page.tsx       # 分析结果页
  components/
    LogInput.tsx          # 日志输入区域
    LogViewer.tsx         # 原始日志展示（高亮）
    AnalysisPanel.tsx     # AI分析结果面板
    TimelineView.tsx      # 时间线视图
    ChatPanel.tsx         # 追问对话面板
  lib/
    parser.ts             # 日志解析器
    detector.ts           # 日志格式检测
    prompts.ts            # LLM prompt模板
    splitter.ts           # 长日志分片策略
```

## 长日志处理策略

- < 4000 tokens: 整段送入
- 4000-20000 tokens: Parser提取ERROR/WARN行+上下文窗口，浓缩后送入
- \> 20000 tokens: 分片分析，LLM二次汇总（二次汇总prompt将各分片的根因结论合并为统一分析）

## UI/UX

```
┌─────────────────────────────────────────────┐
│  LogScope                         [设置图标] │
├─────────────────────┬───────────────────────┤
│                     │                       │
│   原始日志区域       │    AI分析区域          │
│                     │                       │
│   - 高亮ERROR行     │   1. 时间线视图       │

├─────────────────────┴───────────────────────┤

│  > 为什么这个请求失败了？                      │

```

### 交互流程

1. **输入阶段**: 页面顶部大面积文本框 + 拖拽上传区，粘贴/上传后点击"开始分析"
2. **分析阶段**: 左侧日志实时高亮关键行，右侧流式显示分析结果，有进度指示
3. **结果阶段**: 双栏对照浏览，点击左侧行 → 右侧跳到对应分析段落
4. **追问阶段**: 底部对话栏，上下文自动包含当前日志和分析结果

### 交互细节

- 分析进行中，已识别的错误行逐步高亮（不等全部完成）
- 根因置信度用颜色区分：高(绿) / 中(黄) / 低(红)
- 时间线用垂直时间轴可视化，每个节点可展开详情
- 修复建议支持"复制代码"按钮

### 设置面板

- 选择AI模型（OpenAI GPT-4 / Claude 等）
- API Key 输入（存在localStorage，不上传服务端）

## 验证方式

1. 启动 dev server，粘贴一段包含ERROR的真实日志，验证分析流程完整走通
2. 测试不同格式日志（JSON结构化、纯文本、混合），验证格式检测和解析
3. 测试长日志（> 4000 tokens），验证分片策略正确工作
4. 测试追问功能，验证上下文保持正确
5. 测试流式输出，验证结果逐步渲染无卡顿
